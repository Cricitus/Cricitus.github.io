Mysql——Datagrip相关的报错及其解决

net start mysql 后报错为 服务没有报告任何错误

  1. 打开CMD，命令提示符。

  2. 输入netstat -ano，则跳出下图所示，找对对应3306的端口以及其PID

  3. 继续输入tasklist|findstr 端口号对应的PID号，查看具体是什么exe占用了端口

  4. 输入**taskkill /pid PID号 /f **关闭

  5. 重新net start mysql, 问题解决了，可以自行初始化mysql并设置账号密码

备忘: -u root -p 1234

部分傻逼错误

localhost右键新建没有schema等选项，先直接new console，再create schema xxx，之后就有了。

许久之后进入mysql，可能会有以下的报错：

[28000][1045] Access denied for user ‘admin‘@’localhost‘ (using password: YES)

解决方法：

同net start mysql解决方法一样
